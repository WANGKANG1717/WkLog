由于实战经验变得丰富很多，深感日志的重要性，并且学习了解到springboot的日志使用方法
所以我决定自己实现一个简单的日志类，能够实现简单的、足够我日常使用的基本功能
目前决定实现的基本功能有如下几个：
1. 日志格式：
    1. 时间和日期 yyyy-MM-dd HH:mm:ss.SSS
    2. 日志级别：ERROR、WARN、INFO、DEBUG
    3. 消息分隔符：---
    4. 类名和函数名
    5. 消息
2. 文件输出
3. 文件归档和滚动切割
4. 使用properties来进行自定义化配置
    具体可以配置的内容大体有：
        保存日志的路径，日期时间格式，文件归档，滚动切割
5. 大体的思路是，我希望引入log类后，整个程序运行期间，只有一个实例
    log类可以根据调用对象获取对象的名称，然后按照规定的数据格式数据日志


**
一些重要的其他信息
● 由低到高：DEBUG, INFO, WARN, ERROR
  ○ 只会打印指定级别及以上级别的日志
  ○ DEBUG：开发调试细节日志
  ○ INFO：关键、感兴趣信息日志
  ○ WARN：警告但不是错误的信息日志，比如：版本过时
  ○ ERROR：业务错误日志，比如出现各种异常

归档：每天的日志单独存到一个文档中。
切割：每个文件10MB，超过大小切割成另外一个文件。

静默模式：不显示类名和方法名
